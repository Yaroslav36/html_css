# Найбільш поширені помилки в домашніх завданнях з HTML та CSS

## 1. Недостатнє використання семантичних тегів HTML5

**Проблема:**
Студенти часто використовують загальні теги `<div>` та `<span>` замість семантичних тегів HTML5, що погіршує структуру, доступність та SEO-оптимізацію сторінки.

**Рекомендації:**

- Використовуйте `<header>`, `<main>`, `<footer>` для основних структурних елементів
- Використовуйте `<section>`, `<article>` для розділення контенту
- Використовуйте `<nav>` для навігаційних меню
- Використовуйте `<figure>` та `<figcaption>` для ілюстрацій, діаграм, формул
- Використовуйте `<aside>` для додаткової інформації

**Приклад правильного використання:**

```html
<body>
  <header>
    <h1>Назва сайту</h1>
  </header>
  <main>
    <section>
      <h2>Розділ 1</h2>
      <article>
        <h3>Стаття 1</h3>
        <p>Текст статті...</p>
      </article>
    </section>
  </main>
  <footer>
    <p>Контактна інформація</p>
  </footer>
</body>
```

## 2. Змішування інлайн-стилів та зовнішніх CSS

**Проблема:**
Багато студентів використовують атрибут `style` безпосередньо в HTML-елементах замість того, щоб визначати стилі в зовнішньому CSS-файлі, що ускладнює підтримку та оновлення коду.

**Рекомендації:**

- Створюйте окремий CSS-файл для всіх стилів
- Підключайте CSS-файл через тег `<link>` в `<head>`
- Використовуйте класи для стилізації елементів
- Уникайте інлайн-стилів (`style="..."`)

**Приклад правильного використання:**

```html
<!-- HTML файл -->
<head>
  <link rel="stylesheet" href="styles.css" />
</head>
<body>
  <p class="highlighted">Виділений текст</p>
</body>
```

```css
/* CSS файл (styles.css) */
.highlighted {
  background-color: yellow;
  font-weight: bold;
  padding: 5px;
}
```

## 3. Неправильна ієрархія заголовків

**Проблема:**
Студенти часто порушують ієрархію заголовків, наприклад, переходять від `<h1>` одразу до `<h3>`, пропускаючи `<h2>`, що негативно впливає на структуру документа та доступність.

**Рекомендації:**

- Використовуйте заголовки в правильному порядку: `<h1>`, `<h2>`, `<h3>`, `<h4>`, `<h5>`, `<h6>`
- Кожна сторінка повинна мати лише один `<h1>` заголовок
- Не пропускайте рівні заголовків (наприклад, після `<h1>` повинен йти `<h2>`, а не `<h3>`)

**Приклад правильного використання:**

```html
<h1>Основний заголовок сторінки</h1>
<section>
  <h2>Підзаголовок першого рівня</h2>
  <p>Текст...</p>
  <h3>Підзаголовок другого рівня</h3>
  <p>Текст...</p>
</section>
<section>
  <h2>Ще один підзаголовок першого рівня</h2>
  <p>Текст...</p>
</section>
```

## 4. Відсутність мета-тегів та атрибутів для адаптивності

**Проблема:**
Багато студентів забувають додавати важливі мета-теги, особливо для адаптивності та SEO, а також атрибути мови.

**Рекомендації:**

- Завжди додавайте мета-тег viewport для адаптивності
- Додавайте мета-тег description для SEO
- Вказуйте атрибут `lang` для HTML-елемента
- Додавайте атрибут `alt` для зображень

**Приклад правильного використання:**

```html
<!DOCTYPE html>
<html lang="uk">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <meta name="description" content="Опис сторінки для пошукових систем" />
    <title>Назва сторінки</title>
  </head>
  <body>
    <img src="image.jpg" alt="Детальний опис зображення" />
  </body>
</html>
```

## 5. Неконсистентне іменування класів

**Проблема:**
Студенти часто використовують неінтуїтивні або суперечливі назви класів, що ускладнює розуміння коду.

**Рекомендації:**

- Використовуйте описові назви класів, які відображають призначення елемента
- Дотримуйтесь єдиного стилю іменування (camelCase, kebab-case або snake_case)
- Уникайте назв, що описують зовнішній вигляд (наприклад, `.red`, `.big`), натомість використовуйте назви, що описують призначення (`.warning`, `.primary-heading`)
- Використовуйте префікси для групування пов'язаних класів

**Приклад правильного використання:**

```html
<!-- Неправильно -->
<div class="red big">Важливе повідомлення</div>

<!-- Правильно -->
<div class="alert alert-warning">Важливе повідомлення</div>
```

```css
/* Правильно */
.alert {
  padding: 10px;
  border-radius: 5px;
  margin-bottom: 15px;
}

.alert-warning {
  background-color: #fff3cd;
  color: #856404;
  border: 1px solid #ffeeba;
}
```

## 7. Дублювання коду та стилів

**Проблема:**
Студенти часто дублюють однакові стилі для різних елементів замість використання групування селекторів або створення спільних класів.

**Рекомендації:**

- Групуйте селектори з однаковими стилями
- Створюйте базові класи для спільних стилів
- Використовуйте каскадність CSS
- Розділяйте CSS на логічні блоки (базові стилі, компоненти, утиліти)

**Приклад правильного використання:**

```css
/* Неправильно */
h1 {
  color: #333;
  margin-bottom: 20px;
  font-family: Arial, sans-serif;
}

h2 {
  color: #333;
  margin-bottom: 15px;
  font-family: Arial, sans-serif;
}

/* Правильно */
h1,
h2 {
  color: #333;
  font-family: Arial, sans-serif;
}

h1 {
  margin-bottom: 20px;
}

h2 {
  margin-bottom: 15px;
}
```

## 8. Відсутність коментарів у коді

**Проблема:**
Студенти рідко додають коментарі до свого коду, що ускладнює його розуміння та підтримку.

**Рекомендації:**

- Додавайте коментарі для пояснення складних частин коду
- Використовуйте коментарі для розділення CSS на логічні блоки
- Додавайте коментарі перед важливими функціональними блоками HTML

**Приклад правильного використання:**

```html
<!-- Шапка сайту з логотипом та навігацією -->
<header>
  <img src="logo.png" alt="Логотип компанії" />
  <!-- Головне навігаційне меню -->
  <nav>
    <!-- ... -->
  </nav>
</header>
```

```css
/* Базові стилі для всього сайту */
body {
  font-family: Arial, sans-serif;
  line-height: 1.6;
  color: #333;
}

/* Стилі для шапки сайту */
header {
  background-color: #f8f8f8;
  padding: 20px 0;
  border-bottom: 1px solid #ddd;
}

/* Стилі для навігаційного меню */
nav {
  display: flex;
  justify-content: space-between;
}
```

## 9. Неправильне використання семантичних елементів

**Проблема:**
Студенти часто використовують семантичні елементи не за призначенням, наприклад, `<b>` замість `<strong>` або `<i>` замість `<em>`.

**Рекомендації:**

- Використовуйте `<strong>` для важливого тексту, а не `<b>` для жирного шрифту
- Використовуйте `<em>` для виділення з наголосом, а не `<i>` для курсиву
- Використовуйте `<mark>` для виділення тексту, який потрібно привернути увагу
- Використовуйте `<cite>` для цитування джерел
- Використовуйте `<code>` для відображення коду

**Приклад правильного використання:**

```html
<!-- Неправильно -->
<p><b>Увага!</b> Це <i>дуже важливо</i> для вашої безпеки.</p>

<!-- Правильно -->
<p><strong>Увага!</strong> Це <em>дуже важливо</em> для вашої безпеки.</p>

<p>Функція <code>alert()</code> відображає спливаюче вікно.</p>

<p>Як сказав <cite>Альберт Ейнштейн</cite>, "Все відносно".</p>
```

## 10. Проблеми з валідністю HTML

**Проблема:**
Студенти часто допускають помилки, які порушують валідність HTML, наприклад, незакриті теги, неправильні вкладення або відсутні обов'язкові атрибути.

**Рекомендації:**

- Завжди закривайте всі теги
- Перевіряйте правильність вкладення тегів
- Додавайте обов'язкові атрибути (наприклад, `alt` для `<img>`)
- Використовуйте валідатор W3C для перевірки HTML (https://validator.w3.org/)

**Приклад правильного використання:**

```html
<!-- Неправильно -->
<div>
  <p>Текст
  <span>Ще текст</div>
</span>

<!-- Правильно -->
<div>
  <p>Текст</p>
  <span>Ще текст</span>
</div>

<!-- Неправильно -->
<img src="image.jpg">

<!-- Правильно -->
<img src="image.jpg" alt="Опис зображення">
```

## Висновок

Уникнення цих поширених помилок допоможе вам створювати більш якісний, підтримуваний та доступний код. Пам'ятайте, що хороший HTML та CSS код повинен бути:

1. **Семантичним** - використовувати правильні теги для правильних цілей
2. **Структурованим** - мати чітку ієрархію та організацію
3. **Адаптивним** - працювати на різних пристроях та розмірах екрану
4. **Підтримуваним** - легким для розуміння та модифікації
5. **Валідним** - відповідати стандартам W3C

Регулярно перевіряйте свій код на валідність та дотримуйтесь сучасних стандартів веб-розробки.
