# Sticky Позиціонування

## Визначення

**Sticky** позиціонування є гібридом `relative` та `fixed` позиціонування. Елемент поводиться як відносно позиціонований (`position: relative`) до досягнення порогового значення при прокрутці, після чого поводиться як фіксовано позиціонований (`position: fixed`).

```css
.element {
  position: sticky;
  top: 0;        /* Порогове значення для активації "липкої" поведінки */
}
```

## Ключові характеристики

- **Початково в потоці документа**: Елемент починає з поведінки як при відносному позиціонуванні, займаючи нормальне місце в потоці документа.
- **Перемикання на fixed при досягненні порогу**: Коли елемент досягає зазначеної відстані від краю вікна перегляду, він стає "липким" і поводиться як фіксовано позиціонований.
- **Обмеження контейнером**: Елемент залишається "липким" тільки в межах свого батьківського контейнера.
- **Необхідність порогового значення**: Як мінімум одне властивість (`top`, `right`, `bottom` або `left`) має бути встановлено для активації липкої поведінки.

## Як працює sticky позиціонування

1. Спочатку елемент поводиться як відносно позиціонований, залишаючись у нормальному потоці документа.
2. Коли відстань між елементом і краєм вікна перегляду досягає заданого порогового значення (наприклад, `top: 0`), елемент "прилипає" до цього місця і починає поводитися як фіксовано позиціонований.
3. Елемент залишається "липким" доти, доки нижній край його батьківського контейнера не прокрутиться до верхнього краю "липкого" елемента.

## Приклад "липких" заголовків секцій

```html
<div class="container">
  <section>
    <h2 class="sticky-header">Розділ 1</h2>
    <p>...</p>
  </section>
  <section>
    <h2 class="sticky-header">Розділ 2</h2>
    <p>...</p>
  </section>
  <section>
    <h2 class="sticky-header">Розділ 3</h2>
    <p>...</p>
  </section>
</div>
```

```css
.container {
  max-width: 800px;
  margin: 0 auto;
}

section {
  padding: 20px;
  margin-bottom: 30px;
  background-color: #f9f9f9;
  border-radius: 8px;
}

.sticky-header {
  position: sticky;
  top: 0;
  background-color: #3498db;
  color: white;
  padding: 10px;
  margin: -20px -20px 20px;
  border-radius: 8px 8px 0 0;
  z-index: 1;
}
```

### Результат:

Кожен заголовок секції (`h2`) спочатку відображається у верхній частині своєї секції. При прокрутці, коли заголовок досягає верхньої частини вікна перегляду (`top: 0`), він "прилипає" до цієї позиції. Коли нижня межа секції досягає верхньої частини заголовка, заголовок знову приєднується до своєї секції і прокручується вгору, звільняючи місце для наступного "липкого" заголовка.

## Приклад "липкого" бічного меню

```html
<div class="content-wrapper">
  <aside class="sidebar">
    <nav class="sticky-nav">
      <ul>
        <li><a href="#section1">Розділ 1</a></li>
        <li><a href="#section2">Розділ 2</a></li>
        <li><a href="#section3">Розділ 3</a></li>
      </ul>
    </nav>
  </aside>
  <main class="main-content">
    <section id="section1">...</section>
    <section id="section2">...</section>
    <section id="section3">...</section>
  </main>
</div>
```

```css
.content-wrapper {
  display: flex;
  max-width: 1200px;
  margin: 0 auto;
}

.sidebar {
  width: 250px;
  padding: 20px;
}

.sticky-nav {
  position: sticky;
  top: 20px; /* Залишає невеликий проміжок зверху */
}

.main-content {
  flex: 1;
  padding: 20px;
}

section {
  min-height: 80vh; /* Для демонстрації прокрутки */
  margin-bottom: 30px;
}
```

## Підтримка браузерами

Sticky позиціонування підтримується у всіх сучасних браузерах, але має деякі обмеження:

- Internet Explorer не підтримує sticky позиціонування
- Safari вимагає префікс `-webkit-`
- Деякі мобільні браузери мають обмежену підтримку

Для забезпечення максимальної сумісності рекомендується використовувати:

```css
.sticky-element {
  position: -webkit-sticky; /* Для Safari */
  position: sticky;
  top: 0;
}
```

## Коли використовувати sticky позиціонування?

- Для заголовків секцій, які повинні залишатися видимими при прокрутці відповідного контенту
- Для бічної навігації, яка повинна залишатися в полі зору при прокрутці основного вмісту
- Для панелей інструментів, які повинні "прилипати" після певної точки прокрутки
- Для рекламних банерів, які повинні залишатися видимими в рамках певної секції
- Для таблиць з "липкими" заголовками рядків або стовпців

## Переваги

- Поєднує переваги відносного та фіксованого позиціонування
- Елементи залишаються в контексті свого батьківського контейнера
- Відмінно підходить для створення ієрархічних інтерфейсів з "липкими" заголовками
- Зменшує необхідність використання JavaScript для подібної функціональності

## Недоліки

- Обмежена підтримка в старіших браузерах
- Може призвести до несподіваної поведінки, якщо не враховувати обмеження батьківським контейнером
- Іноді важко передбачити точну поведінку при складних компонуваннях
- Може створювати проблеми з продуктивністю на мобільних пристроях при великій кількості "липких" елементів

## Найкращі практики

- **Завжди встановлюйте порогове значення**: Як мінімум одна властивість зміщення (`top`, `right`, `bottom`, `left`) повинна бути встановлена.
- **Враховуйте розмір контейнера**: Переконайтеся, що батьківський контейнер має достатній розмір для прояву "липкої" поведінки.
- **Плануйте z-index**: При використанні кількох "липких" елементів плануйте їх накладання за допомогою `z-index`.
- **Забезпечте поліфіл для старіших браузерів**: Використовуйте JavaScript-бібліотеки для імітації sticky поведінки в браузерах без підтримки.
- **Тестуйте на різних пристроях**: Поведінка може відрізнятися на різних розмірах екранів і браузерах.

## Випадки використання у сучасному веб-дизайні

- **Таблиці даних**: "Липкі" заголовки стовпців або рядків для великих таблиць.
- **Інтерфейси з фільтрами**: Фільтри, які "прилипають" зверху під час перегляду результатів.
- **Деревоподібні структури**: "Липкі" заголовки категорій або груп у довгих списках.
- **Облікові записи товарів**: "Липкі" деталі продукту або кнопка "Додати до кошика" під час перегляду опису товару.
- **Довгі форми**: "Липкі" групи полів або кнопки відправки форми.
