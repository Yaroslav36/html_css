# Fixed Позиціонування

## Визначення

**Fixed** позиціонування дозволяє закріпити елемент відносно вікна браузера (viewport), так що елемент залишається на одному місці навіть при прокрутці сторінки.

```css
.element {
  position: fixed;
  top: 0;        /* Прикріплення до верхнього краю вікна браузера */
  left: 0;       /* Прикріплення до лівого краю вікна браузера */
}
```

## Ключові характеристики

- **Виключення з потоку документа**: Елемент повністю виключається з нормального потоку, не займаючи місця в макеті.
- **Позиціонування відносно вікна браузера**: Елемент позиціонується відносно вікна браузера, а не відносно будь-якого елемента на сторінці.
- **Фіксована позиція при прокрутці**: Елемент залишається на тому ж місці при прокрутці сторінки.
- **Створює новий шар**: Фіксовано позиціоновані елементи рендеряться в новому шарі і можуть перекривати інші елементи.
- **Ігнорує контейнер**: На відміну від `position: absolute`, фіксоване позиціонування ігнорує будь-які предки з не-статичним позиціонуванням.

## Правила зміщення

- `top: 0` - прикріплення до верхнього краю вікна браузера
- `right: 0` - прикріплення до правого краю вікна браузера
- `bottom: 0` - прикріплення до нижнього краю вікна браузера
- `left: 0` - прикріплення до лівого краю вікна браузера

## Приклад фіксованої навігації

```html
<header class="fixed-header">
  <nav>
    <ul>
      <li><a href="#home">Головна</a></li>
      <li><a href="#about">Про нас</a></li>
      <li><a href="#services">Послуги</a></li>
      <li><a href="#contact">Контакти</a></li>
    </ul>
  </nav>
</header>

<main>
  <section id="home">...</section>
  <section id="about">...</section>
  <section id="services">...</section>
  <section id="contact">...</section>
</main>
```

```css
.fixed-header {
  position: fixed;
  top: 0;
  left: 0;
  width: 100%;
  background-color: #fff;
  box-shadow: 0 2px 5px rgba(0, 0, 0, 0.1);
  z-index: 1000;
}

nav ul {
  display: flex;
  list-style: none;
  padding: 0;
  margin: 0;
}

nav li {
  margin: 0 15px;
}

nav a {
  display: block;
  padding: 15px 0;
  text-decoration: none;
  color: #333;
}

main {
  /* Важливо: додаємо відступ зверху для компенсації фіксованого заголовка */
  padding-top: 60px; /* Має бути >= висоти фіксованого заголовка */
}

section {
  height: 100vh; /* Для демонстрації прокрутки */
  padding: 20px;
}
```

### Результат:

Навігаційний заголовок залишається прикріпленим до верхньої частини вікна браузера при прокрутці сторінки, а контент починається нижче фіксованого заголовка (завдяки `padding-top` на `main`).

## Приклад кнопки "Повернутися вгору"

```html
<a href="#top" class="back-to-top">↑</a>
```

```css
.back-to-top {
  position: fixed;
  bottom: 20px;
  right: 20px;
  width: 40px;
  height: 40px;
  background-color: #3498db;
  color: white;
  text-align: center;
  line-height: 40px;
  border-radius: 50%;
  text-decoration: none;
  font-weight: bold;
  box-shadow: 0 2px 5px rgba(0, 0, 0, 0.3);
  opacity: 0.8;
  transition: opacity 0.3s;
}

.back-to-top:hover {
  opacity: 1;
}
```

## Особливості на мобільних пристроях

На мобільних пристроях фіксоване позиціонування може працювати не зовсім так, як очікується:

1. **iOS Safari до версії 13**: Фіксоване позиціонування не працювало правильно при прокрутці (елементи "стрибали").
2. **Клавіатура на мобільних пристроях**: Поява віртуальної клавіатури може зміщувати фіксовані елементи на деяких пристроях.
3. **Трансформації**: Застосування CSS-трансформацій до предків фіксованого елемента може змінити контекст позиціонування.

### Рішення для мобільних пристроїв

```css
/* Використання position: sticky замість fixed для кращої мобільної підтримки */
.mobile-header {
  position: sticky;
  top: 0;
  /* інші стилі... */
}

/* або */

/* Використання -webkit-overflow-scrolling для iOS Safari */
body {
  -webkit-overflow-scrolling: touch;
}
```

## Коли використовувати fixed позиціонування?

- Для навігаційних панелей, які повинні бути завжди видимі
- Для кнопок "повернутися вгору"
- Для модальних вікон, які повинні залишатися в центрі екрана
- Для сповіщень і банерів, які повинні бути постійно видимі
- Для панелей інструментів і віджетів чату

## Переваги

- Елементи завжди видимі користувачеві, незалежно від прокрутки
- Ідеально підходить для навігації та важливих елементів керування
- Не залежить від структури інших елементів на сторінці
- Спрощує доступ до важливих функцій сайту

## Недоліки

- Може створювати проблеми на мобільних пристроях
- Займає постійний простір у вікні перегляду, потенційно зменшуючи видиму область вмісту
- Може перекривати важливий вміст, якщо не враховувати відступи
- Проблеми з доступністю, якщо не враховувати можливість зміни масштабу сторінки

## Найкращі практики

- **Додавайте компенсаційні відступи**: Додавайте відступи до основного вмісту, що дорівнюють висоті фіксованого елемента.
- **Використовуйте z-index розумно**: Переконайтеся, що фіксовані елементи мають належний рівень z-index.
- **Розглядайте альтернативи для мобільних пристроїв**: На мобільних пристроях використовуйте sticky позиціонування або приховуйте фіксовані елементи при прокрутці вниз.
- **Переконайтеся в доступності**: Елементи повинні залишатися доступними при масштабуванні сторінки.

## Наступний крок

Після вивчення фіксованого позиціонування, перейдемо до [липкого позиціонування (sticky)](./sticky-positioning.md), яке поєднує поведінку relative і fixed позиціонування.
