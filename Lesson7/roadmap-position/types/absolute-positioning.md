# Absolute Позиціонування

## Визначення

**Absolute** позиціонування дозволяє розміщувати елемент в точно визначеному місці відносно найближчого предка з не-статичним позиціонуванням (тобто з `position: relative`, `absolute`, `fixed` або `sticky`).

```css
.element {
  position: absolute;
  top: 20px;     /* 20px від верхнього краю найближчого позиціонованого предка */
  left: 30px;    /* 30px від лівого краю найближчого позиціонованого предка */
}
```

## Ключові характеристики

- **Виключення з потоку документа**: Елемент повністю виключається з нормального потоку документа, не займаючи місця в макеті.
- **Позиціонування відносно предка**: Елемент позиціонується відносно найближчого предка з не-статичним позиціонуванням.
- **Якщо немає позиціонованого предка**: Елемент позиціонується відносно початкового блоку контейнера (зазвичай `<html>` або `<body>`).
- **Шар поверх інших елементів**: Абсолютно позиціоновані елементи створюють новий шар і можуть перекривати інші елементи.
- **Розміри за вмістом**: Якщо не вказані явно розміри, ширина і висота визначаються за вмістом елемента.

## Правила зміщення

- `top: 20px` - 20px від верхнього краю позиціонованого предка
- `right: 20px` - 20px від правого краю позиціонованого предка
- `bottom: 20px` - 20px від нижнього краю позиціонованого предка
- `left: 20px` - 20px від лівого краю позиціонованого предка

## Важливість контексту позиціонування

Найбільш поширена помилка при роботі з `position: absolute` - невизначення контексту позиціонування (забуття про `position: relative` для батьківського елементу).

```html
<!-- Неправильно -->
<div class="parent">
  <div class="absolute-child">Абсолютний елемент</div>
</div>

<!-- Правильно -->
<div class="parent relative">
  <div class="absolute-child">Абсолютний елемент</div>
</div>
```

```css
/* Неправильно - дочірній елемент буде позиціонуватися відносно <body> */
.parent {
  width: 300px;
  height: 200px;
  border: 2px solid #3498db;
}

/* Правильно - визначаємо контекст позиціонування */
.relative {
  position: relative;
}

.absolute-child {
  position: absolute;
  top: 10px;
  right: 10px;
  background-color: #e74c3c;
  padding: 10px;
  color: white;
}
```

## Приклад

```html
<div class="container">
  <div class="box box1">Box 1</div>
  <div class="box-container">
    <div class="box box2">Box 2 (absolute)</div>
  </div>
  <div class="box box3">Box 3</div>
</div>
```

```css
.container {
  background-color: #f0f0f0;
  padding: 20px;
}

.box-container {
  position: relative;
  height: 150px;
  background-color: #f9f9f9;
  border: 1px dashed #ccc;
}

.box {
  width: 100px;
  height: 100px;
  margin: 10px;
  background-color: #3498db;
  color: white;
  text-align: center;
  line-height: 100px;
}

.box2 {
  position: absolute;
  top: 20px;
  left: 50%;
  transform: translateX(-50%); /* Центрування елемента */
  margin: 0; /* Видаляємо margin, оскільки елемент абсолютно позиціонований */
  background-color: #e74c3c;
}
```

### Результат:

Box 2 позиціонується абсолютно всередині `.box-container` і розташовується по центру цього контейнера (завдяки `left: 50%` і `transform: translateX(-50%)`). Box 3 розташовується безпосередньо під `.box-container`, ігноруючи абсолютно позиціонований Box 2.

## Розтягування елемента

Однією з унікальних можливостей абсолютного позиціонування є можливість розтягнути елемент, встановивши всі властивості зміщення в 0:

```css
.full-overlay {
  position: absolute;
  top: 0;
  right: 0;
  bottom: 0;
  left: 0;
  background-color: rgba(0, 0, 0, 0.5); /* Напівпрозорий фон */
}
```

Цей код створює елемент, що повністю перекриває свій позиціонований контейнер - ідеально для оверлеїв та модальних вікон.

## Коли використовувати absolute позиціонування?

- Для створення спливаючих підказок (tooltips)
- Для позиціонування UI елементів, які повинні знаходитися над іншим вмістом
- Для модальних вікон та накладань
- Для декоративних елементів, що повинні знаходитися в конкретних позиціях
- Для індикаторів, таких як значки сповіщень або бейджі

## Переваги

- Точний контроль над позиціонуванням елементів
- Можливість накладання елементів один на одного
- Елементи не впливають на потік документа
- Можливість розтягування елемента на весь контейнер

## Недоліки

- Може ускладнити створення адаптивних макетів
- Можливі проблеми з перекриттям тексту та інших елементів
- Вимагає уважного планування контексту позиціонування
- Може створювати проблеми для доступності, якщо використовується неправильно

## Найкращі практики

- Завжди визначайте контекст позиціонування за допомогою `position: relative` на батьківському елементі
- Уникайте абсолютного позиціонування для основних елементів макета - використовуйте сучасні сіткові системи (CSS Grid) або Flexbox
- Використовуйте `transform: translate(-50%, -50%)` разом з `top: 50%` і `left: 50%` для точного центрування
- Перевіряйте, щоб абсолютно позиціоновані елементи коректно працювали на різних розмірах екранів
- Намагайтеся обмежувати використання абсолютного позиціонування для невеликих, декоративних елементів

## Наступний крок

Після вивчення абсолютного позиціонування, перейдемо до [фіксованого позиціонування (fixed)](./fixed-positioning.md), яке позиціонує елементи відносно вікна браузера.
