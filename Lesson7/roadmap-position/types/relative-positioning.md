# Relative Позиціонування

## Визначення

**Relative** позиціонування дозволяє зміщувати елемент відносно його початкового положення в нормальному потоці документа, не впливаючи на розташування інших елементів.

```css
.element {
  position: relative;
  top: 20px;     /* Зміщення вниз на 20px від початкового положення */
  left: 30px;    /* Зміщення вправо на 30px від початкового положення */
}
```

## Ключові характеристики

- **Залишається в потоці документа**: Елемент займає своє місце в нормальному потоці документа, навіть після зміщення.
- **Зміщення відносно початкової позиції**: Властивості `top`, `right`, `bottom`, `left` зміщують елемент відносно його початкового положення.
- **Створює контекст позиціонування**: Стає точкою відліку для дочірніх елементів з `position: absolute`.
- **Впливає на z-index**: Дозволяє контролювати накладання елементів за допомогою `z-index`.

## Правила зміщення

- `top: 20px` - зміщує елемент **вниз** на 20px від початкової позиції
- `right: 20px` - зміщує елемент **вліво** на 20px від початкової позиції
- `bottom: 20px` - зміщує елемент **вгору** на 20px від початкової позиції
- `left: 20px` - зміщує елемент **вправо** на 20px від початкової позиції

## Приклад

```html
<div class="container">
  <div class="box box1">Box 1</div>
  <div class="box box2">Box 2 (relative)</div>
  <div class="box box3">Box 3</div>
</div>
```

```css
.container {
  background-color: #f0f0f0;
  padding: 20px;
}

.box {
  width: 100px;
  height: 100px;
  margin: 10px;
  background-color: #3498db;
  color: white;
  text-align: center;
  line-height: 100px;
}

.box2 {
  position: relative;
  top: 30px;
  left: 50px;
  background-color: #e74c3c;
}
```

### Результат:

Box 2 зміщується на 30px вниз і 50px вправо від свого початкового положення, але при цьому його оригінальне місце в потоці документа залишається зарезервованим. Це означає, що Box 3 відображається так, ніби Box 2 знаходиться на своєму початковому місці.

## Створення контексту позиціонування для дочірніх елементів

Одне з найважливіших застосувань `position: relative` - створення контексту позиціонування для дочірніх елементів з `position: absolute`.

```html
<div class="parent">
  Батьківський елемент
  <div class="child">Дочірній елемент</div>
</div>
```

```css
.parent {
  position: relative;
  width: 300px;
  height: 200px;
  border: 2px solid #3498db;
  padding: 20px;
}

.child {
  position: absolute;
  top: 10px;
  right: 10px;
  background-color: #e74c3c;
  padding: 10px;
  color: white;
}
```

У цьому прикладі дочірній елемент позиціонується відносно батьківського елемента (з `position: relative`), а не відносно всього документа.

## Коли використовувати relative позиціонування?

- Для незначного коригування положення елемента
- Для створення контексту позиціонування для дочірніх елементів з `position: absolute`
- Коли потрібно застосувати `z-index` для контролю накладання елементів
- Коли елемент повинен залишатися в потоці документа, але потрібно змістити його

## Переваги

- Легке зміщення елементів без порушення загальної структури сторінки
- Створення контексту позиціонування для абсолютно-позиціонованих елементів
- Можливість використання `z-index` для контролю накладання
- Елемент залишається в потоці документа

## Недоліки

- Може створювати порожнє місце на сторінці (місце, де елемент повинен був бути)
- Іноді важко передбачити точне положення після зміщення (особливо при адаптивному дизайні)

## Найкращі практики

- Використовуйте `position: relative` переважно для створення контексту позиціонування для дочірніх елементів
- Для незначних зміщень краще використовувати margin або padding
- Не рекомендується використовувати великі значення зміщення, оскільки це може порушити макет на менших екранах
- Не використовуйте протилежні властивості зміщення одночасно (наприклад, `top` і `bottom`)

## Наступний крок

Розглянувши відносне позиціонування, перейдемо до [абсолютного позиціонування (absolute)](./absolute-positioning.md), яке дозволяє повністю виключити елемент з потоку документа.
