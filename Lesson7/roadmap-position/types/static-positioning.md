# Static Позиціонування

## Визначення

**Static** - це значення за замовчуванням для властивості `position`. Елементи з `position: static` розташовуються згідно з нормальним потоком документа.

```css
.element {
  position: static; /* Можна не вказувати, тому що це значення за замовчуванням */
}
```

## Ключові характеристики

- **Поведінка за замовчуванням**: Якщо ви не вказали значення для властивості `position`, елемент буде мати `position: static`.
- **Порядок елементів**: Елементи відображаються зверху вниз, зліва направо (для мов, які читаються зліва направо).
- **Вертикальна орієнтація**: Блокові елементи (`div`, `p`, `h1-h6` тощо) займають всю доступну ширину та розташовуються один під одним.
- **Горизонтальна орієнтація**: Рядкові елементи (`span`, `a`, `strong` тощо) розташовуються один за одним в рядку.
- **Властивості зміщення не діють**: Властивості `top`, `right`, `bottom`, `left` та `z-index` не впливають на елементи зі статичним позиціонуванням.

## Приклад

```html
<div class="container">
  <div class="box box1">Box 1</div>
  <div class="box box2">Box 2 (static, top: 50px - не працює)</div>
  <div class="box box3">Box 3</div>
</div>
```

```css
.container {
  background-color: #f0f0f0;
  padding: 20px;
}

.box {
  width: 100px;
  height: 100px;
  margin: 10px;
  background-color: #3498db;
  color: white;
  text-align: center;
  line-height: 100px;
}

.box2 {
  /* Це не матиме жодного ефекту, оскільки елемент має position: static за замовчуванням */
  top: 50px;
  left: 50px;
  background-color: #e74c3c;
}
```

### Результат:

Box 2 залишиться на своєму вихідному місці, незважаючи на вказані властивості `top` і `left`, оскільки ці властивості не працюють із статичним позиціонуванням.

## Коли використовувати static позиціонування?

- Для більшості елементів сторінки, які повинні слідувати нормальному потоку документа
- Коли вам не потрібно точно позиціонувати елемент або накладати його на інші елементи
- Для дочірніх елементів, які не повинні створювати контекст позиціонування

## Коли не використовувати static позиціонування?

- Коли елемент потрібно розташувати не за нормальним потоком документа
- Коли потрібно змістити елемент відносно його початкового положення
- Коли потрібно створити контекст позиціонування для абсолютно-позиціонованих дочірніх елементів

## Переваги

- Передбачувана поведінка за замовчуванням
- Не вимагає додаткових властивостей
- Добре підходить для основної структури сторінки

## Недоліки

- Обмежена гнучкість у розташуванні елементів
- Неможливість використовувати властивості зміщення
- Не створює контекст позиціонування для дочірніх елементів
