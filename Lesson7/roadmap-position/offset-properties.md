# Властивості зміщення і z-index

Для точного контролю над позиціонуванням елементів CSS надає спеціальні властивості зміщення та властивість управління шарами (z-index).

## Властивості зміщення

Ці властивості визначають, наскільки елемент зміщений від свого початкового або відносного положення.

### Основні властивості зміщення

```css
.element {
  top: value; /* Зміщення від верхнього краю */
  right: value; /* Зміщення від правого краю */
  bottom: value; /* Зміщення від нижнього краю */
  left: value; /* Зміщення від лівого краю */
}
```

### Різниця в поведінці для різних типів позиціонування

| Тип позиціонування | Вплив властивостей зміщення                                          |
| ------------------ | -------------------------------------------------------------------- |
| Static             | Не впливають взагалі                                                 |
| Relative           | Зміщують елемент від його початкового положення                      |
| Absolute           | Зміщують елемент від найближчого позиціонованого предка              |
| Fixed              | Зміщують елемент від країв вікна браузера                            |
| Sticky             | Застосовуються як порогові значення для активації "липкої" поведінки |

### Одиниці виміру для властивостей зміщення

```css
.element {
  top: 10px; /* Піксельні значення */
  right: 10%; /* Відсотки від розміру контейнера */
  bottom: 1em; /* Відносно розміру шрифту елемента */
  left: 1rem; /* Відносно розміру шрифту кореневого елемента */
}
```

### Скорочений запис: властивість `inset`

CSS також пропонує скорочений запис для одночасного встановлення всіх чотирьох властивостей зміщення за допомогою властивості `inset`:

```css
/* Всі сторони мають однакове зміщення */
.element {
  inset: 10px; /* Еквівалент top: 10px; right: 10px; bottom: 10px; left: 10px; */
}

/* Різні значення для вертикальних і горизонтальних сторін */
.element {
  inset: 10px 20px; /* top і bottom: 10px, left і right: 20px */
}

/* Різні значення для всіх сторін */
.element {
  inset: 10px 20px 15px 25px; /* top: 10px; right: 20px; bottom: 15px; left: 25px; */
}
```

### Важливі обмеження

1. **Конфлікт протилежних властивостей**: Уникайте одночасного використання протилежних властивостей (наприклад, `top` і `bottom` або `left` і `right`).

2. **Залежність від розміру контейнера**: При використанні відсотків, значення розраховуються відносно розміру батьківського контейнера.

## Властивість z-index

`z-index` керує порядком накладання елементів по осі Z (від екрана до глядача).

### Основи z-index

```css
.element {
  z-index: 1; /* Цілочисельне значення */
}
```

### Як працює z-index

- За замовчуванням всі елементи мають `z-index: auto` (еквівалентно 0)
- Елементи з більшим значенням `z-index` відображаються над елементами з меншим значенням
- `z-index` працює тільки з не-статичними елементами (з `position` відмінним від `static`)
- Від'ємні значення `z-index` розміщують елементи позаду елементів з автоматичним z-index

### Контекст накладання (stacking context)

Кожен елемент з не-статичним позиціонуванням і `z-index` відмінним від `auto` створює новий контекст накладання. Це означає:

1. Дочірні елементи не можуть бути розташовані під елементами, які нижче їхнього батьківського контексту.
2. Дочірні елементи не можуть бути розташовані над елементами, які вище їхнього батьківського контексту.

```html
<div class="parent1" style="z-index: 1;">
  <div class="child" style="z-index: 1000;">Element A</div>
</div>
<div class="parent2" style="z-index: 2;">
  <div class="child" style="z-index: 1;">Element B</div>
</div>
```

У цьому прикладі Element B буде над Element A, навіть незважаючи на те, що Element A має більший z-index. Це тому, що Element B є частиною контексту накладання з вищим z-index.

### Рекомендації щодо використання z-index

1. **Використовуйте розумну систему значень**:

   ```css
   :root {
     --z-dropdown: 100;
     --z-sticky-header: 200;
     --z-modal: 300;
     --z-tooltip: 400;
     --z-notification: 500;
   }

   .dropdown {
     z-index: var(--z-dropdown);
   }
   .sticky-header {
     z-index: var(--z-sticky-header);
   }
   ```

2. **Мінімальні потрібні значення**: Використовуйте найменше можливе значення, яке досягає бажаного ефекту (наприклад, 1, 2, 3 замість 100, 200, 300).

3. **Розумно використовуйте контексти накладання**: Розуміння контекстів накладання допоможе уникнути несподіваної поведінки елементів.

4. **Документуйте використання**: Створіть документацію про вашу систему z-index, особливо для великих проєктів.

## Приклад: Комплексне позиціонування з використанням властивостей зміщення і z-index

```html
<div class="card">
  <div class="card-ribbon">Special</div>
  <img src="product.jpg" alt="Product" />
  <div class="card-content">
    <h3>Product Title</h3>
    <p>Product description...</p>
  </div>
  <button class="card-button">Buy Now</button>
</div>
```

```css
.card {
  position: relative;
  width: 300px;
  height: 400px;
  border-radius: 8px;
  box-shadow: 0 4px 8px rgba(0, 0, 0, 0.1);
  overflow: hidden;
}

.card-ribbon {
  position: absolute;
  top: 20px;
  right: -30px;
  background-color: #e74c3c;
  color: white;
  padding: 5px 30px;
  transform: rotate(45deg);
  z-index: 2;
}

.card img {
  width: 100%;
  height: 200px;
  object-fit: cover;
}

.card-content {
  padding: 20px;
}

.card-button {
  position: absolute;
  bottom: 0;
  left: 0;
  right: 0;
  padding: 15px;
  background-color: #3498db;
  color: white;
  border: none;
  z-index: 1;
}
```

## Сучасні альтернативи прямому позиціонуванню

У сучасному веб-розробленні для багатьох завдань макетування краще використовувати Flexbox або CSS Grid замість властивостей позиціонування:

```css
/* Центрування з Flexbox замість абсолютного позиціонування */
.container {
  display: flex;
  justify-content: center;
  align-items: center;
  height: 100vh;
}

/* Складний макет з CSS Grid замість абсолютного позиціонування */
.grid-layout {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  grid-template-rows: auto 1fr auto;
  grid-template-areas:
    "header header header"
    "sidebar main main"
    "footer footer footer";
  height: 100vh;
}

.header {
  grid-area: header;
}
.sidebar {
  grid-area: sidebar;
}
.main {
  grid-area: main;
}
.footer {
  grid-area: footer;
}
```
